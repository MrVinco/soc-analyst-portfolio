## 1. Executive Summary

This report documents the analysis of a PCAP file from Malware-Traffic-Analysis.net. The investigation identified repeated HTTP POST traffic from an internal host to an external IP address.

The internal host `10.2.28.88` repeatedly contacted `45.131.214.85` using the URI `/fakeurl.htm`. The traffic used the User Agent `NetSupport Manager/1.3` and showed command style values such as `CMD=POLL`, `CMD=ENCD`, and encoded `DATA` fields in the TCP stream.

The repeated timing, same destination, same URI, small POST bodies, and NetSupport Manager User Agent are consistent with command and control beaconing over HTTP. No clear evidence of bulk data exfiltration was confirmed in the POST stream.

## 2. Investigation Scope

The purpose of this analysis was to:

* Identify the main internal host in the PCAP
* Review DNS, HTTP, and TCP traffic
* Identify suspicious external communications
* Follow suspicious TCP streams
* Extract indicators of compromise
* Determine whether traffic showed C2 beaconing, DNS tunneling, or data exfiltration
* Map findings to MITRE ATT&CK
* Provide defensive recommendations

## 4. Initial Traffic Review

### Protocol Hierarchy

The PCAP contained multiple protocol types, including:

* DNS
* HTTP
* TLS
* SMB / NetBIOS
* LDAP
* Kerberos
* DHCP

This indicated a Windows network environment with domain related traffic and web communication.

### Conversations Review

The IPv4 conversations showed that `10.2.28.88` was the main internal host communicating with both internal and external systems. The largest internal communication was between `10.2.28.88` and `10.2.28.2`.

Several external IP addresses appeared in the conversation list, but the strongest suspicious activity was later identified through HTTP analysis.

## 5. DNS Analysis

### DNS Verdict

No clear DNS tunneling was confirmed. The DNS queries observed were not unusually long or random looking, and there was no obvious high volume encoded DNS activity.

## 6. HTTP Analysis

Filtering for HTTP requests revealed suspicious repeated POST traffic.

### Suspicious HTTP Request

| Field          | Value                              |
| -------------- | ---------------------------------- |
| Source IP      | 10.2.28.88                         |
| Destination IP | 45.131.214.85                      |
| Method         | POST                               |
| URI            | /fakeurl.htm                       |
| Full URL       | hxxp://45.131.214[.]85/fakeurl.htm |
| User Agent     | NetSupport Manager/1.3             |
| Content Type   | application/x-www-form-urlencoded  |

### TCP Stream Findings

Following the TCP stream showed repeated HTTP POST requests to the same endpoint.

The stream contained command style values such as:

```text
CMD=POLL
CMD=ENCD
DATA=
```

The server responded with HTTP `200 OK`.

This behavior is consistent with a host polling an external command and control endpoint.

## 7. Beaconing Analysis

TShark was used to extract timestamps for HTTP POST requests from `10.2.28.88` to `45.131.214.85`.

The output showed repeated POST requests to the same destination and URI over a long period.

### Beacon Characteristics

| Indicator                 | Observation                                        |
| ------------------------- | -------------------------------------------------- |
| Same internal host        | 10.2.28.88                                         |
| Same external destination | 45.131.214.85                                      |
| Same URI                  | /fakeurl.htm                                       |
| Same User Agent           | NetSupport Manager/1.3                             |
| Repeated timing           | Approximately every 60 seconds after initial burst |
| Request body size         | Mostly small content length of 36 bytes            |

### Beaconing Verdict

C2 beaconing is confirmed. The repeated timing pattern, consistent destination, repeated URI, and command style TCP stream strongly suggest command and control communication over HTTP.

## 8. Data Exfiltration Review

TShark output showed most POST requests had small request bodies.

Common values observed:

| Field               | Common Value |
| ------------------- | ------------ |
| Frame length        | 286 bytes    |
| HTTP content length | 36 bytes     |

This does not indicate large outbound transfer activity. A data exfiltration case would usually show larger POST bodies, large file transfers, FTP upload activity, or sustained high outbound traffic.

### Data Exfiltration Verdict

Data exfiltration was not confirmed in this stream. The traffic is better classified as C2 beaconing or command polling.

## 11. Defensive Recommendations

1. Isolate the internal host `10.2.28.88` for investigation.
2. Block outbound communication to `45.131.214[.]85`.
3. Hunt for additional traffic using the User Agent `NetSupport Manager/1.3`.
4. Review endpoint logs on `10.2.28.88` for remote access tool execution.
5. Check proxy, firewall, and DNS logs for connections to `/fakeurl.htm`.
6. Create detection logic for repeated HTTP POST requests to the same external IP at regular intervals.
7. Alert on unexpected NetSupport Manager traffic where the tool is not approved.
8. Perform malware triage or full forensic review of the affected host.
9. Reset credentials used on the host if compromise is confirmed.
10. Review other hosts for similar C2 behavior.

## 12. Detection Opportunities

Potential SOC detections from this case:

### Detection 1: Repeated HTTP POST Beaconing

Alert when one internal host sends repeated HTTP POST requests to the same external IP and URI at regular intervals.

### Detection 2: Suspicious NetSupport Manager User Agent

Alert when `NetSupport Manager/1.3` appears in HTTP traffic from hosts where NetSupport is not approved.

### Detection 3: Repeated POST to Raw External IP

Alert when HTTP POST requests are made directly to an IP address instead of a domain name.

### Detection 4: Small Periodic POST Requests

Alert when POST requests have small content length and repeat at consistent intervals.

## 14. Conclusion

The PCAP analysis identified repeated HTTP POST traffic from `10.2.28.88` to `45.131.214.85` using `/fakeurl.htm`. The traffic used the User Agent `NetSupport Manager/1.3` and occurred at roughly regular intervals.

The evidence supports a finding of command and control beaconing over HTTP. The POST bodies were small, so data exfiltration was not confirmed from this stream.

The affected host should be treated as compromised and isolated for further endpoint investigation.
