## 1. Executive Summary

This incident report documents three simulated credential based attacks in a Windows SOC home lab. The attacks were password guessing, password spraying, and credential stuffing. The simulations were performed against local Windows test accounts inside a controlled virtual machine environment.

Splunk was used to detect the activity through Windows Security Event Logs. The main observed event IDs were 4625 for failed logon attempts, 4624 for successful logon activity, and 4740 for account lockout. All detections were mapped to MITRE ATT&CK technique T1110 Brute Force and its relevant sub techniques.

## 5. Detection Details

### BF 001 Password Guessing

The password guessing simulation generated rapid failed login attempts against one account, testuser1. Splunk detected 10 failed logon attempts within a five minute window.

Detection logic:

A password guessing alert is triggered when five or more failed logons occur against the same username within a short time window.

Evidence file:

```text
screenshots/BF-001-password-guessing.png
```

### BF 002 Password Spray

The password spraying simulation attempted one common password against multiple accounts. Splunk detected failed logons across testuser1, testuser2, and testuser3 within the same time window.

Detection logic:

A password spray alert is triggered when failed logon attempts are spread across three or more unique usernames within a ten minute window.

Evidence file:

```text
screenshots/BF-002-password-spray.png
```

### BF 003 Credential Stuffing

The credential stuffing simulation generated failed attempts against testuser2 before a successful login occurred. Splunk detected three failures followed by one success for the same account.

Detection logic:

A credential stuffing alert is triggered when failed logons are followed by a successful login for the same username within a thirty minute window.

Evidence file:

```text
screenshots/BF-003-credential-stuffing.png
```

### BF 004 Account Lockout

The account lockout detection identified Event ID 4740 after repeated failed attempts against testuser1. This provided high confidence evidence of repeated authentication failure against one account.

Detection logic:

An account lockout alert is triggered when Event ID 4740 appears in the Windows Security Event Log.

## 7. Containment Actions

The activity occurred in a lab environment and did not affect any production systems.

Actions taken:

1. Confirmed all test activity was limited to the Windows 10 lab VM.
2. Verified that testuser1 was locked after repeated failed attempts.
3. Unlocked testuser1 after analysis.
4. Confirmed testuser2 successful login was part of the controlled credential stuffing simulation.
5. Preserved screenshots and SPL detection logic for documentation.

## 8. Recommendations

1. Enable multi factor authentication for all user accounts.
2. Monitor Event ID 4625 for repeated failed logons.
3. Monitor Event ID 4740 as a high confidence account lockout signal.
4. Detect password spraying by counting distinct usernames targeted from the same source.
5. Detect credential stuffing by correlating failed logons followed by successful logons.
6. Tune lockout thresholds to reduce both account compromise risk and denial of service risk.
7. Use scheduled Splunk alerts instead of real time alerts in small lab environments to reduce CPU load.

## 9. Lessons Learned

This lab showed that credential attacks do not all look the same in logs. Password guessing creates depth against one user, password spraying creates breadth across many users, and credential stuffing creates a sequence of failures followed by a successful login. Separating these patterns improves detection accuracy and helps analysts respond with better context.

## 10. Conclusion

The lab successfully simulated and detected password guessing, password spraying, credential stuffing, and account lockout activity using Splunk and Windows Security Event Logs. The detections were mapped to MITRE ATT&CK T1110 sub techniques and documented with screenshots, SPL files, simulation scripts, and an incident report.
