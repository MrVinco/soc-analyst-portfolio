## Executive Summary

An alert was generated for encoded PowerShell command execution on the Windows 10 lab host.

The activity was part of an authorized detection engineering simulation. The simulation used a harmless encoded PowerShell command to validate Script Block Logging, Splunk log ingestion, and the PS-001 detection rule.

The detection fired successfully. No malicious activity was identified.

## Alert Trigger

The alert triggered when PowerShell Event ID 4104 recorded encoded command activity.

The simulation used the following suspicious PowerShell execution patterns:

* `-EncodedCommand`
* `-NoProfile`
* `-NonInteractive`
* `-WindowStyle Hidden`

These flags are commonly reviewed during SOC investigations because attackers may use them to hide or automate PowerShell execution.

## Investigation Details

### Step 1: Confirmed Log Source

The alert came from the expected PowerShell Operational log source:

```text
WinEventLog:Microsoft-Windows-PowerShell/Operational
```

The event was recorded as:

```text
EventCode=4104
```

This confirms Script Block Logging captured the PowerShell activity.

### Step 2: Reviewed Script Block Content

The script block showed a harmless simulation message:

```text
SIMULATION: Encoded command executed
```

No malicious payload, credential access, persistence, or destructive behavior was observed.

### Step 3: Confirmed Lab Context

The command was executed inside the Windows 10 lab VM as part of the Suspicious PowerShell Detection Engineering project.

The activity matched the expected simulation behavior.

### Step 4: Reviewed Risk

Encoded PowerShell commands can be suspicious because attackers may use encoding to hide commands from basic review.

In this case, the encoded command was harmless and used only to validate detection coverage.

## Lessons Learned

The PS-001 detection rule successfully identified encoded PowerShell activity in Splunk.

This confirmed that:

* Script Block Logging was enabled
* PowerShell Operational logs were collected by Splunk
* Event ID 4104 was searchable
* Encoded PowerShell activity could be detected
* The detection rule worked as expected

## Recommended Analyst Actions for Real Incidents

If this alert appeared in a real environment, the analyst should:

1. Decode the encoded command.
2. Review the full script block content.
3. Identify the user and host.
4. Check the parent process using Sysmon Event ID 1.
5. Check for related network connections.
6. Review Defender events.
7. Search for the same command across other hosts.
8. Escalate if download, evasion, persistence, or credential access behavior is observed.
