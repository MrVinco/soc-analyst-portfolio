# PA-2026-001: Phishing Email Analysis

## Case Summary

This report documents a phishing email analysis exercise completed in a safe Windows virtual machine environment.

The analysis focused on identifying email artifacts, reviewing message headers, inspecting an attached email file, extracting indicators of compromise, safely previewing a suspicious URL, and mapping findings to MITRE ATT&CK.

Exact challenge answers and solved screenshots are not included because this lab is based on active training platform content.

## Objective

The objective of this case was to investigate a suspicious email forwarded to a SOC analyst and identify useful artifacts such as:

* Recipient details
* Subject and timestamp
* Originating IP evidence
* Reverse DNS information
* Attachment name
* Suspicious URL
* Hosting service
* Safe webpage preview result

## Tools Used

| Tool                | Purpose                                        |
| ------------------- | ---------------------------------------------- |
| Thunderbird         | Opened and reviewed the `.eml` file safely     |
| Sublime Text        | Searched raw email headers and message content |
| URL2PNG             | Safely previewed the suspicious webpage        |
| WHOIS / DomainTools | Checked reverse DNS information                |
| VirusTotal          | Intended for IOC reputation review             |
| AbuseIPDB           | Intended for IP reputation review              |

## Methodology

The email was first opened in Thunderbird to review the visible message content and attachment structure. Thunderbird was useful for understanding the email layout, visible subject, recipient information, and attached message file.

Sublime Text was then used to inspect the raw email content. This helped identify technical fields such as originating IP evidence, received headers, PHP script metadata, and suspicious URL strings.

The suspicious URL was not opened directly in a browser. It was checked using URL2PNG to safely preview the webpage without interacting with the live site.

## Findings Summary

The email showed characteristics commonly seen in phishing or spam activity. It used a delivery failure theme and contained an attached original email message. The attached content included suspicious promotional text and an external URL hosted on a public blogging platform.

The raw email headers also showed web form related metadata, suggesting that a public website contact form may have been abused to send the message.

## Analyst Notes

This case showed the importance of using the right tool for each part of phishing analysis. Thunderbird was better for safely reading the email and viewing the attachment structure. Sublime Text was better for raw header inspection and searching technical fields.

The analysis also reinforced the importance of defanging URLs, avoiding direct interaction with suspicious links, and documenting evidence clearly.
